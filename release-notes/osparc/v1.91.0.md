# Release Notes

### Version 1.91.0 · April 09, 2026

Better communication. Smoother workflows. More resilient services.

This release improves how the platform communicates with users, brings a cleaner experience in the workbench, and adds resilience to service state tracking.

---

## ✨ Highlights

### 📰 In-App Release Notes

New version popups are now more informative.

When a new platform version is released, instead of showing a basic message with a link, the **"New Version Released" dialog now renders the full release notes directly inside the popup**. The markdown content is fetched and styled inline, so you can read what changed without leaving the app.


### 📂 Improved Files Tab in the Workbench

The "Storage/Files" panel on the left side of the workbench has been restructured.

It now presents a **combined view showing both the current project's files and other available storage locations** in a single tree. A new reload button makes it easy to refresh the file listing without leaving your workflow.

### 🎞️ App Mode Editor: Cleaner & Auto-Saving

The App Mode Editor has been redesigned:

- **Auto-save** — changes are persisted automatically, removing the need for a manual "Save" button
- **Clearer visual cues** for which steps have instructions assigned
- **Instruction windows** now properly close when leaving App Mode
- Bug fix: selecting a node in the editor now correctly adds it to the slideshow

### 🔄 More Resilient Service State Tracking

No more forever Pending state.

Interactive services no longer get stuck showing a "pending" state when a WebSocket notification is lost. A **periodic REST polling fallback** has been added so the frontend will always catch up with the actual service state, even if the real-time channel drops a message.

---

## 🛠 Fixes & Stability Improvements

This release also includes:
- Fixed a race condition in the file picker progress patching (RTC)
- Fixed active filters not being shown correctly in the dashboard
- Prevented users from duplicating projects that are currently open
